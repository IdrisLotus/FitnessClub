# Веб-приложение для учета клиентов фитнес-клуба

  **Дипломный проект на Django для автоматизации учета клиентов, абонементов, посещений, оплат, тренеров и расписания занятий фитнес-клуба**


---

## О проекте

Веб-приложение разработано в рамках выпускной квалификационной работы на тему **«Разработка веб-приложения для учета клиентов фитнес-клуба»**.

Приложение предназначено для сотрудников фитнес-клуба и позволяет заменить ручной учет в таблицах или бумажных журналах на единую информационную систему с авторизацией, разграничением доступа и понятным веб-интерфейсом.

## Возможности

### Работа с пользователями

- Авторизация через встроенную систему Django
- Выход из системы
- Профиль пользователя
- Разграничение доступа по ролям
- Основные роли: администратор и тренер

### Учет клиентов

- Добавление клиентов
- Просмотр списка клиентов
- Редактирование данных клиента
- Удаление клиента
- Поиск и фильтрация записей
- Хранение контактной информации и статуса клиента

### Учет тренеров

- Добавление тренеров
- Просмотр списка тренеров
- Редактирование информации о тренере
- Удаление тренера
- Хранение специализации и контактных данных

### Абонементы

- Создание абонементов для клиентов
- Учет срока действия абонемента
- Учет количества посещений
- Отображение активных и завершенных абонементов
- Контроль доступности абонемента при посещениях

### Посещения

- Ведение журнала посещений
- Регистрация посещения клиента
- Связь посещения с клиентом и абонементом
- Просмотр истории посещений
- Учет даты и времени посещения

### Оплаты

- Регистрация платежей клиентов
- Учет суммы и даты оплаты
- Связь оплаты с клиентом и абонементом
- Просмотр истории платежей

### Расписание занятий

- Создание занятий
- Назначение тренера на занятие
- Указание даты, времени и количества мест
- Запись клиентов на занятия
- Проверка свободных мест
- Запрет повторной записи одного клиента
- Отмена записи
- Отметка статуса посещения занятия

### Отчеты

- Просмотр сводной информации по работе фитнес-клуба
- Отчеты по клиентам
- Отчеты по посещениям
- Отчеты по оплатам
- Отчеты по абонементам

## Установка и запуск

### Требования

- Python 3.x
- Git
- pip
- Visual Studio Code или другой редактор кода

### Запуск проекта

```bash
# Клонировать репозиторий
git clone https://github.com/IdrisLotus/FitnessClub.git
cd FitnessClub

# Создать виртуальное окружение
python -m venv .venv

# Активировать виртуальное окружение
# Windows:
.venv\Scripts\activate

# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install -r requirements.txt

# Выполнить миграции
python manage.py migrate

# Создать администратора
python manage.py createsuperuser

# Запустить сервер разработки
python manage.py runserver
```

После запуска проект будет доступен в браузере по адресу:

```text
http://127.0.0.1:8000/
```

## База данных

На этапе разработки используется SQLite. Данные хранятся в локальном файле `db.sqlite3`, который не добавляется в репозиторий GitHub.

Для создания таблиц базы данных после клонирования проекта необходимо выполнить:

```bash
python manage.py migrate
```

Для доступа к административной панели нужно создать суперпользователя:

```bash
python manage.py createsuperuser
```

---

## Административная панель

Django предоставляет встроенную административную панель. После создания суперпользователя она доступна по адресу:

```text
http://127.0.0.1:8000/admin/
```

Через административную панель можно управлять записями моделей проекта, если они зарегистрированы в `admin.py` соответствующих приложений.

## Проверка работоспособности

Для проверки проекта можно использовать следующие команды:

```bash
# Проверка настроек проекта
python manage.py check

# Проверка миграций
python manage.py makemigrations --check

# Запуск сервера
python manage.py runserver
```

После запуска сервера необходимо открыть приложение в браузере и проверить основные сценарии работы: вход в систему, добавление клиента, создание абонемента, регистрация посещения, добавление оплаты, создание занятия и просмотр отчетов.
